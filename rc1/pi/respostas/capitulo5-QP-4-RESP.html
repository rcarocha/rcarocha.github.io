Questão 1

Qual é o objetivo do endereço MAC?

Opções

  1. especificar um identificador único por estação para ser utilizado pela camada de enlace para identificar remetentes e destinatários de mensagens de enlace.
  2. especificar um identificador único por estação para ser utilizado pela camada de rede para enviar mensagens para estações na rede local.
  3. identificar fabricante e número de série de estações (hardware).
  4. especificar um endereço fixo para estações em redes locais.

Questão 2

Qual a relação entre o endereço IP de uma estação e o seu endereço MAC?

Opções

  1. ambos os endereços identificam a estação, sendo o IP para a camada de rede e o MAC para a camada de enlace.
  2. em uma rede local, os endereços IP e MAC de uma estação devem ser os mesmos.
  3. nenhuma em particular, mas ambos os endereços devem ser utilizado para camada de enlace para enviar as suas mensagens.
  4. duas estações podem ter o mesmo endereço IP, mas não podem ter o mesmo endereço MAC.
  5. duas estações podem ter o mesmo endereço MAC, mas não podem ter o mesmo endereço IP.

Questão 3

Em que situação duas estações na rede podem ter (A) mesmo endereço IP ou (B) mesmo endereço MAC?

Opções

  1. (A) nunca, exceto se for privado - (B) desde que as estações garantidamente estejam em redes locais diferentes
  2. (A) nunca, exceto se for privado - (B) nunca
  3. tanto em (A) e (B): desde que as estações estejam garantidamente em redes locais diferentes.
  4. tanto em (A) e (B): desde que as estações estejam garantidamente em redes IP diferentes.

Questão 4

Qual o significado do endereço MAC de broadcast?

Opções

  1. É um endereço para o qual a camada de enlace entregará pacotes para todas as estações na rede local.
  2. É um endereço para o qual a camada de rede entregará pacotes para todas as estações na rede local.
  3. É um apelido, na camada de enlace, para o endereço de broadcast da camada de rede.
  4. É a soma de todos os endereços da camada de enlace.
  5. É a soma de todos os endereços da camada de enlace na rede local.

Questão 7

Qual é o objetivo do protocolo ARP?

Opções

  1. descobrir o endereço MAC de uma estação na rede local, a partir seu endereço IP.
  2. descobrir o endereço MAC de uma estação qualquer na internet dado o seu endereço IP.
  3. descobrir o endereço IP de uma estação na rede local, a partir seu endereço MAC.
  4. descobrir o endereço IP de uma estação qualquer na internet dado o seu endereço MAC.

Questão 8

Por que o protocolo ARP é necessário para uma comunicação de uma estação?

Opções

  1. Porque o protocolo IP, para solicitar o envio de uma mensagem pela camada de enlace para uma estação na rede local, precisa saber qual endereço MAC está associado àquele IP.
  2. Porque o protocolo da camada de enlace, para solicitar o envio de uma mensagem pela camada de rede para uma estação na rede local, precisa saber qual endereço IP está associado a um endereço MAC particular.
  3. Porque o protocolo da camada de enlace, para solicitar o envio de uma mensagem pela camada de rede para uma estação FORA da rede local, precisa saber qual endereço IP está associado a um endereço MAC particular.
  4. Para a camada de enlace especificar qual será o identificador único de cada estação.
  5. Porque em redes locais, a entrega das mensagens ocorre via camada de enlace e, assim como ocorre com o DNS para o protocolo IP, o ARP permite à camada de enlace descobrir o endereço MAC das estações, sem o qual não é possível transmitir mensagens.

Opções

  1. o endereço MAC de DESTINO seja 00:1D:7D:B2:34:F9 ou FF:FF:FF:FF:FF:FF.
  2. o endereço MAC de ORIGEM seja 00:1D:7D:B2:34:F9 ou FF:FF:FF:FF:FF:FF.

  4. unicamente se o endereço MAC de ORIGEM for 00:1D:7D:B2:34:F9.
  5. unicamente se o endereço MAC de DESTINO for 00:1D:7D:B2:34:F9 e o endereço IP do pacote IP for igual ao endereço IP da estação.

Questão 14

  5. ele enviará um novo ARP request APENAS se o endereço IP da estação mudar.
