Resposta das Questões Pré-aula

QP-3.3-1: Qual das frases abaixo melhor define o que é um socket?

  • Construção de programação para permitir a programas enviar e receber mensagens diretamente pela camada de transporte.
  • API de programação para esconder as chamadas de sistemas operacional como socket(), bind() e listen().
  • Conjunto de primitivas para manipular o TCP e UDP.
  • Protocolos da camada de aplicação que permitem aos programas enviar e receber mensagens pela rede.
  • Conjunto de classes Java que permitem enviar e receber mensagens pela rede.

QP-3.3-2: Qual a relação entre sockets e protocolos da camada de transporte?

  • Os sockets são uma interface com os protocolos da camada de transporte (TCP e UDP), permitindo receber e enviar mensagens por eles.
  • Os sockets são alternativas ao uso dos protocolos da camada de transporte (TCP e UDP).
  • Os sockets implementam os protocolos da camada de transporte.
  • Os sockets oferecem os mesmos serviços que são oferecidos pelos protocolos TCP e UDP.
  • Todas as demais afirmativas estão corretas.

QP-3.3-3: Dois processos precisam conversar entre si e ambos precisam tanto enviar como receber mensagens (comunicação bidirecional). Então serão necessários ...

  • um socket para cada processo, sendo que um é socket servidor e outro é socket cliente.
  • dois sockets para cada processo (quatro no total), sendo que cada um precisa de um socket cliente para transmitir mensagens e um servidor para receber mensagens.
  • um socket para os dois processos, que deve ser criado no processo cliente.
  • um socket para os dois processos, que deve ser criado no processo servidor.
  • um socket para os dois processos, se o socket for TCP (um única conexão), ou um para cada processo (dois no total), se forem sockets UDP.

QP-3.3-4: O que ocorre na chamada de sistema listen()?

  • Socket servidor solicita a espera por conexões em uma determinada porta.
  • Socket servidor fica bloqueado na chamada até que chegue um pedido de comunicação (conexão TCP) de um cliente, quando então o socket aceita o pedido e o canal de comunicação se estabelece.
  • Socket cliente conecta a um socket servidor (conexão TCP), estabelecendo um canal de comunicação onde ambos podem enviar e receber bytes.
  • Associa um número de porta a um socket.

QP-3.3-5: O que ocorre na chamada de sistema accept()?

  • Socket servidor fica bloqueado na chamada até que chegue um pedido de comunicação (conexão TCP) de um cliente, quando então o socket aceita o pedido e o canal de comunicação se estabelece.
  • Socket servidor solicita a espera por conexões em uma determinada porta.
  • Socket cliente conecta a um socket servidor (conexão TCP), estabelecendo um canal de comunicação onde ambos podem enviar e receber bytes.
  • Associa um número de porta a um socket.

QP-3.3-6: O que ocorre na chamada de sistema bind()?

  • Associa um número de porta a um socket.
  • Socket servidor fica bloqueado na chamada até que chegue um pedido de comunicação (conexão TCP) de um cliente.
  • Socket servidor solicita a espera por conexões em uma determinada porta.
  • Socket cliente conecta a um socket servidor (conexão TCP), estabelecendo um canal de comunicação onde ambos podem enviar e receber bytes.

QP-3.3-7: O que ocorre na chamada de sistema connect()?

  • Socket cliente conecta a um socket servidor (conexão TCP), estabelecendo um canal de comunicação onde ambos podem enviar e receber bytes.
  • Socket cliente conecta a um socket servidor (TCP ou UDP), estabelecendo um canal de comunicação onde ambos podem enviar e receber bytes.
  • Socket servidor fica bloqueado na chamada até que chegue um pedido de comunicação (conexão TCP) de um cliente.
  • Socket servidor solicita a espera por conexões em uma determinada porta.
  • Associa um número de porta a um socket.
