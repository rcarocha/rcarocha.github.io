Questão 1

Duas estações estão ligadas na mesma rede local. A entrega do pacote de rede de uma para a outra usa qual tipo de roteamento?

Opções

  1. Direto
  2. Direto e estático
  3. Direto e dinâmico
  4. Indireto estático
  5. Indireto dinâmico
  6. Não envolve roteamento

Questão 2

O que diferencia o roteamento estático do dinâmico?

Opções

  1. No estático, as tabelas de roteamento mantém rotas fixas, as rotas variam ao longo do tempo.
  2. No estático, os destinatários devem estar na mesma rede local dos remetentes das mensagens.
  3. No roteamento dinâmico, dois pacotes de uma mesma origem para um mesmo destino podem ser enviados por caminhos distintos e no estático os pacotes sempre seguem pelo mesmo caminho.
  4. No dinâmico, os possíveis destinatário só são conhecidos quando os pacotes chegam, enquanto que o estático já sabe a priori todos os possíveis destinatários.

Questão 5

Quando chega um pacote, como o protocolo IP determina para qual protocolo de transporte ele deve ser entregue?

Opções

  1. O IP entrega para o protocolo associado ao número indicado no campo Protocol.
  2. O IP entrega para a camada de transporte que identifica o protocolo correto ao inspecionar o campo de dados.
  3. Cada pacote já está associado a uma conexão TCP ou conversa UDP.
  4. Pelo campo de Identification do pacote.

Questão 6

O checksum do pacote IP tem qual propósito?

Opções

  1. Verificar a corretude do cabeçalho do pacote.
  2. Verificar a corretude do campo de dados do pacote.
  3. Verificar a corretude de todo o conteúdo do pacote.
  4. Verificar a corretude da rota por onde o pacote está sendo enviado.
  5. Verificar a consistência com o checksum do UDP e/ou TCP.

Opções

  2. Número de caminhos (rotas) em que o pacote poderá passar.
  3. Segundos
  4. Millisegundos.
  5. Bytes.
  6. O significado depende da estação transmissora.
  7. O significado depende da estação destino do pacote.

Questão 8

Por que os pacotes IP tem tempo de vida?

Opções

  1. Para evitar que pacotes fiquem circulando indefinidamente na rede, na ocasião em que, por qualquer motivo, o pacote não possa ser entregue a um destino.
  2. Para evitar que ocorra o timeout dos pacotes.
  3. Para limitar os atrasos dos pacotes na rede.
  4. Para estabelecer o tempo de atraso que deve ser considerado pelas estações de destino do pacote.
  5. Para estabelecer o tempo de atraso que deve ser considerado pela estaççao de origem do pacote, para efeito de recebimento de ACKs.

Questão 11

Qual é o motivo do IP precisar fragmentar os pacotes?

Opções

  1. Um pacote IP precisa transportar um segmento de transporte no seu campo de dados que não cabe no tamanho máximo do pacote (quadro) na camada de enlace.
  2. A camada de rede recebe uma stream (sequência não estruturada de bytes) da camada de transporte e precisa quebrá-la em pacotes para realizar o transporte.
  3. Roteadores não tem espaço em memória para processar todo um pacote e precisa ajustá-lo à memória disponível.
  4. Dois pacotes IPs precisam ser roteados (encaminhados) em rotas diferentes.
  5. Uma conexão TCP precisa ser fragmentada em diversos pacotes para serem transmitidos pela camada de rede.

Questão 12

Onde ocorre a remontagem de fragmentos do IP?

Opções

  1. Na estação de destino do pacote.
  2. Em cada roteador.
  3. No roteador seguinte àquele que fragmentou o pacote.
  4. No último roteador antes da estação de destino.
  5. No roteador que recebe o fragmento com MF=0 (para um número de pacote com fragmentações).
  6. Na origem do pacote.

Questão 13

Quando chega um pacote com MF=1, qual ação o IP realiza?

Opções

  1. Armazena o pacote, que trata-se de um fragmento, de aguarda a chegada dos demais fragmentos para então remontá-los.
  2. O IP conclui que é o primeiro fragmento de um pacote, e aguarda a chegada dos demais para remontá-lo.
  3. Identifica que o pacote não está fragmentado e então envia para a camada de transporte.
  4. Identifica que é o último fragmento esperado e realiza a remontagem do pacote.

Questão 14

Qual das situações abaixo não representa uma situação em que um pacote pode ser descartado no roteador?

Opções

  1. Um dos fragmentos de pacotes não chegou até o timeout.
  2. O checksum do cabeçalho do pacote está incorreto.
  3. O campo TTL ficou com o valor zero.
  4. Não há memória para processar (armazenar) o pacote.
