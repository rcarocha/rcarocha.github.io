  • A resposta do servidor HTTP (porta 80) disponível no endereço 193.25.185.36, e a barra de endereços do navegador informará o endereço acessado como http://193.25.185.36/.

QP-3.2-5: Por que o HTTP informa o número da versão do protocolo na própria requisição (e também na resposta)?

  • Para indicar ao servidor a versão mais recente do HTTP que o cliente é capaz de compreender e então, ambos conversarem com a versão mais recente em que ambos são compatíveis.
  • Para indicar ao servidor que o cliente espera conversar utilizando o protocolo HTTP.
  • Para indicar qual porta deve ser utilizada para comunicação, de acordo com o protocolo do cliente, que no caso é 80, já que o protocolo é HTTP.
  • Para forçar o servidor a utilizar conexões persistentes.
  • Na verdade, a versão do protocolo é interpretada pelo TCP (HTTP usa TCP!) para que o TCP saiba para qual processo os pacotes recebidos devem ser entregues, que são processos que "falam" HTTP na versão indicada.

QP-3.2-9: Um cliente precisa fazer várias requisições a um servidor para carregar todo o conteúdo de uma página. Assinale a afirmativa verdadeira.

  • *Para permitir o carregamento mais rápido do conteúdo, o cliente HTTP precisará abrir diversas conexões TCP simultâneas com o mesmo servidor. *
  • Para permitir o carregamento mais rápido do conteúdo, o cliente HTTP precisará enviar diversas requisições em pipeline.
  • Se o servidor ou cliente não permitirem conexões persistentes, então será possível realizar o carregamento simultâneo de conteúdo.
  • Se o cliente precisar estabelecer diversas conexões TCP simultâneas com o mesmo servidor, será necessário que este possua diversas portas servidoras disponíveis, além da porta 80.
  • O cliente só poderá realizar as várias requisições se o servidor utilizar HTTP versão 1.1 ou em uma versão mais recente.
